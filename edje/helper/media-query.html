Media Query

We don't like using mixin for media query, instead we use variable within `@media` block like this: ```scss .button +h( p1 ) @media ($below-s) +h( p05 ) @media ($above-m) and ($portrait) +h( p15 ) ``` Those variables are defined in **_settings.scss**. Let me explain what those mean. First thing first, there are 4 default breakpoints in Edje: - XS (extra xmall) is 480px - S (small) is 767px - M (medium) is 960px - and L (large) is 1120px. So the variable `$below-s` means when the screen is below 767px or translated to `max-width: 767px`. Same thing goes to `$above-m` which translated to `min-width: 961px`. For "$above", we add 1px to the breakpoint so it won't clash with "$below".

Available Variables

New Size

If you want to add your own media-query size, simply create new variable in settings. ```scss $below-watch : "max-width: 220px"; $above-watch : "min-width: 221px"; ``` ```scss @media ($below-watch) ... ``` -----

Between Two Sizes

Simply combine two variables using `and`. The example below will target media query between 768px to 960px. ```scss @media ($above-s) and ($below-m) ... ``` -----

One-Time Use

When you only need to use the size once, we provide a function so you don't need to create a new variable. ``` below( $size ) above( $size ) ``` **EXAMPLE** ```scss @media (below(512px)) ... @media (above(753px)) ... ``` -----

Print and Screen

There is no need for fancy variables when specifying Print / Screen media query. ```scss @media only print ... @media only screen and ($below-s) ... ```
